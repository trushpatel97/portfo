004 : Breached Sites

Summary

Websites are meant to be secured. Or at least we hope... however, hackers can bypass network security and steal local files. This is what we call a "breached site". It may seem like hackers are only targetting companies, but that's not the case. Companies have one thing valuable that belongs to you...your data. Any data you provided to a company, has a posibility to be leaked.

This project will let a user enter the url of a site, and output whether the site has been breached or not. If breached, it will give a description of how the breach occurred. If the user wants to see more specific information, such as type of data leaked (passwords, address, phone number, etc.) then they may do so as well. Or the user can request to see everything about the breach. This project uses PWNED's API to operate.

Contribution

  • Pulled data from a JSON file
  • Search for a breached site, that haveibeenpwned.com does not offer
  • Allowed user to have more control to see specifics of a breach
