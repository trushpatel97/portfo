003 : Vet Database

Summary

One of the most important things in a business, is data. More specifically, data of customers/patients. In this project, I made a MS SQL vet database that updates, deletes, and adds data to the database. Each pet, vet, and appointment have an unique key. Of course, each pet will have more fields such as name, DOB, last visit, address, etc. However, when a receptionist wants to schedule an appointment for a pet, they need something to set each pet apart from one another. For example, if we want to schedule an appointment for a pet, and two pets can live in the same address, we need to differentiate the two pets so we can schedule an appointment for the appropriate pet. The receptionist should be able to use an unique ID to see more data if needed.

All that being said, databases can be boring to work with. However, I made a GUI that communicates from Java to the SQL database. Say I want to schedule an appointment for pet # 200, instead of manually adding it to the database table, my GUI allows you to give the pet an appointment_ID, appointment_date, and the vet it will be assigned to. Now say, the receptionist made a mistake by giving the appointment to pet # 200 instead of pet #201. What they can do is simply update the field with my update button. Finally, if the receptionist wants to delete the entire appointment altogether, they can simply enter the appointment_ID and delete that appointment.
